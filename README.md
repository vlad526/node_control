Встановлення та запуск локально

1. Клонуй репозиторій

2
Переконайся, що в тебе встановлено Node.js (LTS версія 22).
Завантажити її можна з офіційного сайту Node.js.

3
Всі необхідні пакети (як для продакшну, так і для розробки) вже прописані у package.json

виконай:npm install

4
Створи конфігураційний файл .env у корені проєкту на основі файлу-прикладу:

.env.example

Відкрий файл .env та заповни необхідні змінн

5
npm run seed — Заповнення бази даних тестовими/базовими даними (дозволи, дефолтні ролі).

npm run lint — Запуск перевірки коду лінтером.

npm run start:dev — Запуск сервера у режимі розробки

npm run start — Запуск продакшн-збірки.

6
Postman-колекція

Файл колекції знаходиться тут: postman/cars-api.postman_collection.json

Як імпортувати колекцію в Postman:
Відкрий програму Postman.

Натисни кнопку Import (у верхньому лівому куті).

Обери файл cars-api.postman_collection.json з папки postman.

Колекція з’явиться у твоєму списку робочих областей.

7
Змінні середовища
доступні такі змінні:

{{API_HOST}} — URL сервера (наприклад, http://localhost:3000).

{{buyerToken}} — JWT токен покупця.

{{sellerToken}} — JWT токен продавця.

{{managerToken}} — JWT токен менеджера.

{{adminToken}} — JWT токен адміністратора.

{{id}} — Ідентифікатор користувача або авто для запитів GET/PUT/DELETE.

{{carId}} — ID автомобіля для процесів верифікації модератором.

Після реєстрації або успішного логіну скрипти Postman (вкладка Tests) автоматично збережуть токен у відповідну змінну середовища.
